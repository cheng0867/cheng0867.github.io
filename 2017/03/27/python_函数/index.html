作用域

即代码的作用范围

LEGB规则（优先顺序）

locals -> enclosing -> globals -> builtins

  • locals：局部变量，当前所在命名空间（如函数、模块），其中：函数的参数也属于命名空间内的变量
  • enclosing：外部嵌套函数的命名空间（如闭包）
  • globals：全局变量，函数定义所在模块的命名空间
  • builtins：内建模块的命名空间

生成器

一边循环一边计算的机制，称为生成器 generator

  • 生成器记住上一次返回时在函数体中的位置。对生成器函数的第二次（或第N次）调用跳转至该函数中间，而上次调用的所有局部变量都保持不变
  • 生成器不仅“记住”了数据状态，还“记住”了在流控制构造中的位置

生成器特点

  1. 节约内存
  2. 迭代到下一次的调用时，所使用的参数都是第一次所保留下的，即是说，在整个所有函数调用的参数都是第一次所调用时保留的，而不是新创建的
  3. 生成器也是可迭代对象，可以使用for迭代

迭代器

可以被next()函数调用并不断返回下一个值的对象称为迭代器 Iterator

迭代器特点

  • 迭代：访问集合元素的一种方式
  • 迭代器：可以记住遍历位置的对象
  • Iterable类型：凡是可作用于for循环的对象
  • Iterator类型：凡是可作用与next()函数的对象
  • iter()函数：集合数据类型如list、dict、str等是Iterable，但不是Iterator，可用iter()函数获得一个Iterator对象

闭包

内部函数对外部函数作用域里(非全局)变量的引用（非全局变量），则称内部函数为闭包

闭包优点

闭包缺点

装饰器

编写代码原则

  • 封闭：封装已实现的功能代码块，尽量不做更改
  • 开放：在原有封装基础上扩展开发

装饰器解释

  1. 定义装饰函数（闭包）
  2. 使用@语法糖在被装饰函数上加装饰器
  3. 在调用被装饰函数时Python解释器将该函数作为装饰函数的参数传递到装饰函数的闭包函数中
  4. 最终输出结果为传递到闭包函数中调用的被装饰函数，可在调用之前添加“装饰”代码

装饰器功能

  1. 引入日志
  2. 函数执行时间统计
  3. 执行函数前预备处理
  4. 执行函数后清理功能
  5. 权限校验等场景
  6. 缓存
