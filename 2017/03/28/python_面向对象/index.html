类

del()方法

当创建对象后，python解释器默认调用init()方法；当删除一个对象时，python解释器也会默认调用一个方法del()

原理

  • 当一个变量保存了对象的引用，此对象的引用计数加1
  • 当使用del删除变量指向的对象时，如果对象的计数引用为3，就会让这个引用计数减1（此时为2），当引用计数为0时，此时真正删除对象

拷贝

==、is

  • ==比较两个对象是否相等
  • is比较两个引用（内存指向地址）是否相等

深拷贝、浅拷贝（爬虫使用深拷贝）

浅拷贝

拷贝引用，但不拷贝内容，即拷贝对象和被拷贝对象之间有关联

深拷贝

  1. 需要导入模块，import copy
  2. 拷贝：c = copy.deepcopy(a)
  3. 引用的内存地址不同

区别

  • 深拷贝：把内容拷贝放入一片新内存空间（缺点：占内存）
  • 浅拷贝：只拷贝引用（缺点：修改一个值，另外一个也改变，存在安全隐患）

copy.copy()

  • 可变类型：创建新地址，类似深拷贝
  • 不可变类型：类似浅拷贝

私有化

  1. xx：公有变量
  2. _x：私有化属性或方法(约定)
    • from some import *不可导入
    • 类对象和子类可以访问
  3. __xx：避免与子类的属性命名冲突
    • 无法在外部直接访问（即实例对象不能访问）
    • 子类不能继承和访问
    • 注：如果在子类中赋值，那么会在子类定义与父类同名的__xx
  4. __xx__：魔法属性或方法
    • 例如：__init__()
    • 注：不要自己创建此类属性和方法
  5. xx_：用于避免与python关键字冲突

垃圾回收

小整数对象池：（-5, 257）

  • python为了优化速度，使用了小整数对象池，避免为整数频繁申请和销毁内存空间
  • 小整数池中的对象都是python已经建立好的，不会被垃圾回收
  • 在python程序中，所有位于这个范围内的整数使用的都是同一个对象
  • 单个字母同理
  • 定义2个相同的字符串，引用计数为0，触发垃圾回收

大整数对象池

每个大整数，均创建一个新的对象

垃圾回收——整数和字符串总结

  • 小整数（-5， 257）共用对象，常驻内存
  • 单个字符共用对象，常驻内存

  • 单个单词，不可修改，默认开启intern机制，共用对象，引用计数为0，则销毁

  • 字符串（含有空格），不可修改，没开启intern机制，不共用对象，引用计数为0，销毁

  • 大整数不共用内存，引用计数为0，销毁

  • 数值类型和字符串类型在Python中都是不可变的，这意味着你无法修改这个对象的值，每次对变量的修改，实际上是创建一个新的对象

GC垃圾回收（Garbage collection）

python采用的是引用计数机制为主，标记-清除和分代收集两种机制为辅的策略

引用计数机制：

引用计数优点

  • 简单
  • 实时性：如果没有引用，内存就直接释放了。不用像其他机制等到特定时机。实时性还带来一个好处：处理回收内存的时间分摊到了平时

引用计数缺点

  • 维护引用计数消耗资源
  • 循环引用

当list1与list2存在相互引用，如果不存在其他对象对它们的引用，list1与list2的引用计数也仍然为1，所占用的内存永远无法被回收，这将是致命的。循环引用导致内存泄漏，注定python还将引入新的回收机制（标记清除和分代收集）

GC负责任务

  • 为新生成的对象分配内存
  • 识别哪些垃圾对象
  • 从垃圾对象那回收内存

gc模块

导致引用计数+1的情况

  • 对象被创建，例如a=23
  • 对象被引用，例如b=a
  • 对象被作为参数，传入到一个函数中，例如func(a)
  • 对象作为一个元素，存储在容器荣，例如list1=[a,a]

导致引用计数-1的情况

  • 对象的别名被显示销毁，例如 del a
  • 对象的别名被赋予新的对象， 例如a=24
  • 一个对象离开它的作用域，例如f函数执行完毕时，func函数中的局部变量（全局变量不会）
  • 对象所在的容器被销毁，或从容器中删除对象

查看一个对象的引用计数

可以查看a对象的引用计数，但是比正常计数大1，因为调用函数的时候传入a，这会让a的引用计数+1

循环引用导致内存泄漏

引用计数的缺陷是循环引用的问题

执行f2()时，进程占用的内存会不断增大

  • 创建了c1，c2后这两块内存的引用计数都是1，执行c1.t=c2和c2.t=c1后，这两块内存的引用计数为2
  • 在del c1后，内存1的对象的引用计数变为1，由于不是为0，所以内存1的对象不会被销毁，所以内存2的对象的引用数依然是2，在del c2后，同理，内存1的对象，内存2的对象的引用数都是1
  • 虽然它们两个的对象都是可以被销毁的，但是由于循环引用，导致垃圾回收器都不会回收它们，所以就会导致内存泄露

range函数

语法结构

  • start：计数从start开始。默认是从0开始。例如：range(5)等价于range(0, 5)
  • stop：到stop结束，但不包括stop。例如：range(0, 5)是[0, 1, 2, 3, 4]没有5
  • step：每次跳跃的间距，默认为1.例如：range(0, 5)等价于range(0, 5, 1)

返回迭代值：

  • py2使用xrange()
  • py3使用range()
