数据结构和算法作用：
提升运行效率，降低开销，提高性能（解决性能瓶颈）

算法

什么是算法：算法是独立存在的一种解决问题的方法和思想
其中：算法是计算机处理信息的本质

算法五大特性

  1. 输入：算法具有0个或多个输入（即可有可无）
  2. 输出：算法至少有1个或多个输出（即必须有个结果）
  3. 有穷性：算法在有限的步骤之后会自动结束而不会无限循环，并且每一个步骤可以在可接受的时间内完成
  4. 确切性：算法中每一步都有确定的含义，不会出现二义性
  5. 可行性：算法的每一步都是可行的，每一步都能够执行有限的次数完成

算法效率衡量

运算步骤——（决定）——>时间复杂度——（决定）——>算法效率

注：不能单靠程序的运行时间判断效率，不同环境下运行结果不同

时间复杂度

  1. 每一个基本操作的时间————固定的一个时间单位
  2. 多少个基本操作————花费多少时间单位
  3. 该方法忽略机器环境的客观影响

最坏时间复杂度

分析算法时，存在的几种可能：

  • 算法完成工作最少需要多少基本操作，即最优时间复杂度，没有参考价值
  • 算法完成工作最多需要多少基本操作，即最坏时间复杂度，提供了一种保证，表明算法在此种成都的基本操作用一定能完成工作
  • 算法完成工作平均需要多少基本操作，即平均时间复杂度，对算法的全面评价，反应了这个算法的性质

时间复杂度计算规则

  1. 基本操作，即只有常数项，认为其时间复杂度为O(1)
  2. 顺序结构，时间复杂度按加法进行计算（基本操作之间的累加）
  3. 循环结构，时间复杂度按乘法进行计算（低次幂全部去掉，如6n^3+2n^2+3n+4,结果为n3）
  4. 分支结构，时间复杂度取最大值（条件判断中，哪个分支步骤最多就取哪条分支的值）
  5. 判断一个算法的效率时，只关心操作数量的最高次项，次要项和常数项可以忽略
  6. 在没有特殊说明时，我们所分析的算法的时间复杂度都是指最坏时间复杂度

数据结构

概念：数据结构就是基本数据（str，int，float）的封装

高级数据类型（已封装基本类型数据）*：list、tuple、dict、set

算法与数据结构区别

  • 算法：关注解决问题的思路
  • 数据结构：关注处理问题的数据的存储方式
  • 数据的存储方式不同，需要进行处理的算法也不同

什么是程序

程序 = 算法 + 数据结构

抽象数据类型（ADT即Abstract Data Type）

把原有的基本数据，和数据所支持的基本操作，放到一起形成一个整体，就是抽象数据类型

常用数据运算

  • 插入（append，insert）
  • 删除（pop，del）
  • 修改
  • 查找（index，contain）
  • 排序（sort）

顺序表

按顺序连续存储，并以表形式存在

顺序表的基本形式

在内存中（32位系统，最小标识单位：1个字节=8byte），int（整型）在内存中占4个字节
例如：列表中存储多个整型数据，下标表示偏移量

顺序表元素外置

每个单元存储地址，地址指向实际数据存储空间

优点：可存储不同类型数据

顺序表的结构

顺序表由两部分组成

  1. 表头信息
    • 容量（最大数量）
    • 元素个数（当前已存在数量）
  2. 数据区

顺序表的基本实现方式

一体式结构

特点：按顺序排列
优点：结构连续，查找直接
缺点：扩展性不好，扩展时需要重新申请内存空间，表头地址随之改变

分离式结构

特点：数据区为一个地址引用，指向真实数据存储空间
优点：扩展时只需要改变数据区的引用地址，表头地址不改变
缺点：查找数据多一步操作

元素存储区扩充（动态顺序表）

两种策略

  1. 固定增加：如每次扩充增加时，固定增加10个存储空间
    • 特点：节省空间，但操作频繁
  2. 倍增：如扩增时，第一次增加4个，第二次增加4*4个，每次翻倍
    • 特点：操作次数少，但浪费空间，即以空间换时间，推荐方式

顺序表的操作

增加元素

三种方式

  1. 尾部追加元素，时间复杂度O(1)
  2. 非保序插入元素，时间复杂度O(1)
  3. 保序插入元素，时间复杂度O(n)

删除元素

三种方式

  1. 尾部删除元素，时间复杂度O(1)
  2. 非保序删除元素，时间复杂度O(1)
  3. 保序删除元素，时间复杂度O(n)

Python中的顺序表

在Python中的list和tuple采用了顺序表的实现技术，具有所有顺序表性质

list的基本实现技术

  • 下标：使用下标索引查找，时间复杂度O(1)
  • 分离式存储：增加元素的过程中，表对象的表示（使用函数id得到的值）不变
  • 元素外置：可添加不同类型数据
  • 动态数据：数据区随着数据的添加而动态变化
    • 扩充策略：存储区满，以4倍扩充，当超过阀值5万，以1倍扩充

链表

扩充时，不改变原有数据，每次增加元素时，增加新的空间
一维空间的一根线串联起来的数据组成的表，叫做链表

注：顺序表和链表统称为线性表

链表结构

每个数据（节点）由两部分组成

  1. 数据区：当前节点存储的数据
  2. 链接区（指针区）：下一个链接节点的内存地址

单链表

单链表结构

  • 方向：只有一个方向，即从开头到结尾
  • 位置：由一个保存位置的数据指向整个链表
  • 节点：由表元素和下一节点链接两部分组成
  • 开头：第一个结点叫做头节点
  • 结尾：最后一个结点叫做尾结点，尾结点下一链接指向空（0）

单链表的操作

  • is_empyt()链表是否为空
  • length()链表长度
  • travel()遍历整个链表
  • add(item)链表头部添加元素
  • append(item)链表尾部添加元素
  • insert(pos, item)指定位置添加元素
  • remove(item)删除节点
  • search(item)查找节点是否存在

栈操作

树

特点

  • 每个节点都有0个或多个子节点
  • 没有父节点的节点称为根节点
  • 每一个非根节点有且只有一个父节点
  • 除了根节点外，每个子节点可以分为多个不相交的子树

术语

  • 节点的度：一个节点含有的子树个数称为该节点的度
  • 树的度：一棵树中，最大的节点的度称为树的度
  • 叶节点或终端节点：度为0的节点（即再没有子节点）
  • 父节点：一个节点含有子节点，则这个节点称为子节点的父节点
  • 子节点：
  • 兄弟节点：
  • 节点的层次：从根开始定义起，根为第1层，根的子节点为第2层，以此类推
  • 树的高度或深度：树中节点的最大层次
  • 堂兄弟节点：父节点在同一层的节点互为堂兄弟
  • 节点的祖先：从根到该节点所经分支上的所有节点
  • 子孙：以某节点为根的子树中任一节点，都称为该节点的子孙
  • 森林：由m颗不相交的树组成的森林

种类

  • 无序树：任一节点的子节点之间没有顺序的树，也称为自由树

二叉树

定义

性质

  1. 在二叉树的第i层上之多有2^(i-1)个节点 (i>0)
  2. 深度为k的二叉树最多有2^k-1个节点 (k>0)
  3. 对于任意一颗二叉树，如果其叶节点数为N0，而度数为2的节点总数为N2，则N0=N2+1
  4. 具有n个节点的完全二叉树的深度必为log2(n+1)
  5. 完全二叉树，从上至下，从左到右编号，则编号为i的节点，其左子节点编号为2i，右子节点编号为2i+1；双音编号必为i/2 (i=1时为根，除外)
  • 层次遍历：从上至下，从左至右
  • 先序遍历：先根，再左，再右
  • 中序遍历：左，根，右
  • 后续遍历：左，右，根
