综合梳理JavaScript常见知识点

数据类型

number

JavaScript不区分整数和浮点数，统一用number表示

字符串

""和''本身只是一种表示方式，不是字符串的一部分

布尔值

==和===

==：自动转换数据类型，再比较，容易产生错误结果
===：不会自动转换数据类型，如果数据类型不一致，返回false，如果一致，再比较

注：不要使用==比较，始终坚持使用===比较

NaN

NaN是number中的一个特殊，与其它值都不相等，包括它自己

唯一能判断NaN的方法是通过isNaN()函数

因为计算机对浮点数精确计算存在误差。所以要比较两个浮点数是否相等，只能计算它们之差的绝对值，看是否小于某个阈值

null和undefined

null：表示一个空的值
undefined：表示值未定义
0：表示一个数值
""：表示长度为0的字符串

null与undefined两者区分不大，大多数情况使用null，undefined只在判断函数参数是否传递的情况下使用

数组(类似python中的list)

js数组可以包含任意数据类型

两种声明方式

  • 直接声明

  • 使用Array()函数实现

变量保存任意数据类型

命名规则

  • 大小写英文
  • 数字
  • $或_组合
  • 不能用数字开头

示例

动态语言 与 静态语言 区别

  • 动态语言：同s一变量，可以反复赋值，而且可以是不同类型的变量
  • 静态语言：在定义变量时必须指定变量类型，如果赋值时类型不匹配，就会报错

strict模式

  • 全局变量：如果一个变量没有通过var声明就被使用，那么该变量就自动被声明为全局变量
  • 局部变量：使用var声明的变量被限制在变量被声明的函数体内，同名变量在不同的函数体内互不冲突

在strict模式下，强制通过var声明变量，未使用var声明变量就使用的，将导致运行错误

注：为避免这一缺陷，所有的js代码都应该使用strict模式

js的字符串就是用''或""括起来的字符表示

如果字符串内包含'或"，可以使用转义字符\来标识

箭头函数与匿名函数区别：

箭头函数完全修复了this的指向，this总是指向词法作用域，也就是外层调用者obj

如果使用箭头函数，以前的那种hack写法就不需要了

var obj = {
birth:1990,
getAge:function(year){
var b = this.birth; // 1990
var fn = (y) => y - this.birth; // this.birth仍是1990
return fn.call({birth:2000}, year);
}
};
obj.getAge(2015); // 25

标准对象

在JS的世界里，一切都是对象

为了区分对象类型，我们用typeof操作符获取对象的类型，它总是返回一个字符串

类型

  • number
  • string
  • boolean
  • function
  • undefined
  • object
    • null
    • Array

包装对象

number、boolean和string都有包装对象。包装对象用new创建

虽然看上去和原来的值一模一样，但是类型已经变为object了，所以，包装对象和原始值用===比较会返回false

所以不要使用包装对象，尤其是针对string类型

遵守规则总结

  • 不要使用new Number()、new Boolean()、new String()创建包装对象
  • 用parseInt()或parseFloat()来转换任意类型到number
  • 用String()来转换任意类型到string，或者直接调用某个对象的toString()方法
  • 通常不必把任意类型转换为boolean再判断，因为可以直接写if (myVar) { … };
  • typeof操作符可以判断出number、boolean、string、function和undefined
  • 判断Array要使用Array.isArray(arr)
  • 判断null请使用myVar === null
  • 判断某个全局变量是否存在用typeof window.myVar === ‘undefined’
  • 函数内部判断某个变量是否存在用typeof myVar === ‘undefined’
  • 任何对象都有toString()方法，除了null和undefined
  • number对象调用toString()，需要特殊处理
