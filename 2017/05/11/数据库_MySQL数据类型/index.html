MySQL的数据类型

数据类型

数值类型

  • 整数
    • int类型
  • 浮点数
    • float类型：单精度浮点数值
    • double类型：双精度浮点数值
    • decimal类型：精确浮点数值

字符串类型

  • char：固定长度字符串
  • varchar：可变长度字符串，指定值为最大值
  • test：存储较大的文本
  • blob：二进制，一般用于存储图像，声音

日期和时间类型

数据类型选择

  • 允许值
  • 存储效率
  • 格式和显示要求
  • 数据处理要求
  • 排序、索引和比较要求

运算符

算数运算符

  • +：加法
  • -：减法
  • *：乘法
  • /：除法（如果除数为0，结果为null）
  • %：取余

注：如果用字符串表示的数字，用在运算中，将转换为数字的值
如果字符串和数字混合不能正确转换，则转换为0

比较运算符

对表达式的左和右进行比较，结果为1（真）,0（假）或null（不确定）

默认不区分大小写，使用binary关键字区分

逻辑运算符

逻辑运算符可以测试1个或多个表达式的逻辑有效性

not：取反

and：和

or：或

函数

数学函数

  • abs()：绝对值
  • rand()：随机数
  • round(x,y)：返回参数x的四舍五入的有y位小数的值
  • pow(x,y)：返回x的y次幂
  • createst(x1,x2,x3...)：返回集合中最大的值
  • least(x1,x2,x3,...)：返回集合中最大的值

聚合函数

聚合函数是对表中的数据进行集中概括的，经常用在包含group by从句的select查询中

  • avg(col)：指定列平均值
  • count(col)：指定列非null值的个数
  • min(col)：指定列的最小值
  • max(col)：指定列的最大值
  • sum(col)：指定列的所有值之和
  • group_concat(col)：由属于一组的列值连接组合而成的结果

字符串函数

length()：获取一个字符串的长度

trim()：去除指定格式

concat()：连接参数成一个字符串

日期时间函数

now()：获取当前的日期和时间，以YYYY-MM-DD HH:MM:SS的格式返回

curdate()和curtime()：单独获取日期和时间

week()：返回指定日期是一年的第几个星期
yearweek()：返回指定日期是哪一年的第几个星期

加密函数

  • encode(str,key)：使用key作为密匙加密字符串str
  • decode(str,key)：使用key作为密匙解密字符串str
  • md5()：计算字符串str的MD5校验和
  • password(str)：返回字符串str的加密版本

子查询

where/having查询

MySQL可以在where子句或having子句中包含子查询
也可在having子句中的比较运算符前使用子查询，从而用它对父查询创建的组进行过滤

in/exists/from查询

in运算符：检测结果中是否存在某个特定的值，如果检测成功就执行外部的查询
exists运算符：用来查询一个子查询是否产生了结果，只有在返回结果为真时才执行外部查询
from：在一个select语句的from子句中，可以把子查询的结果作为一个表

事物处理

一个事物由作为一个单独单元的一个或多个SQL语句组成，这个单元中的每个SQL语句是互相依赖的，而且单元作为一个整体是不可分割的
如果单元中的一个语句不能成功完成，整个单元都会回滚，所有影响到的数据将返回到事物开始之前的状态
因此，事物中的所有语句都被成功的执行，这个事物才成功执行

事物4个特性

  • 原子性：每个事物都必须被认为是一个不可分割的单元
  • 一致性：不管事物是完全成功还是中途失败，当事物使系统处于一致的状态时存在一致性
  • 孤立性：每个事物在它自己的空间发生，和其它发生在系统中的事物隔离，而且事物的结果值有在它完全被执行时才能看到
  • 持久性：即使系统崩溃，一个提交的事物仍在坚持

事物声明周期

  • 使用start transaction命令来标记一个事物的开始，start transaction命令后跟随的是组成事物的SQL语句
  • 使用begin或begin work命令来初始化一个事物
  • 使用commit命令标记了事物块的结束，把整个事物保存在磁盘上，
  • 使用rollback命令撤销所有变化，如果事物包括事物表和非事物表的变化，非事物表的事物处理部分是不能使用rollback命令撤销的

控制事物行为

  • autocommit变量：自动提交，默认情况下，MySQL的SQL查询一旦被执行，就会自动向数据库提交结果，如果设置set autocommit=0，随后表的更新不会被保存
  • transaction isolation level变量：事物孤立级，MySQL默认为repeatable read孤立级，可使用set修改

事物和性能

为不向系统添加不适当的负担，使用小事物，两个普遍的策略

  1. 保证所有要求的用户输入在发出start transaction命令执勤啊都是可行的
  2. 尝试把大的事物分成小的事物然后分别执行
  3. 选择合适的孤立级，孤立级越高，性能越低，所以选择合适的孤立级，有助于性能油画
  4. 避免死锁，在一个事物环境中，当两个或多个处于不同序列的客户同时想更新相同的数据时，就会发生死锁，应避免发生死锁
