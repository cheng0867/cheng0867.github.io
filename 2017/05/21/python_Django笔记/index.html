中间件

浏览器向服务器发出请求的过程

  1. 将请求报文，封装成HttpRequest
  2. 进行url解析
  3. 调用视图，将request对象传入
  4. 调用模版，完成渲染
  5. 在视图中return结果

中间件在没有侵入源代码的情况下，干预、影响整个处理过程的某个环节，即IOC（MVC模式中使用），面向切面编程

分页

Django提供了数据分页的类，定义在django/core/paginator.py中

Paginator对象

用于对大量数据进行分页处理

  • ‘方法init(列表,int)’：返回分页对象，参数为列表数据，int表示每页数据的条数
  • ‘属性count’：总共有多少条数据
  • ‘属性num_pages’：表示页面的总数
  • ‘属性page_range’：页面的索引，从1开始
  • ‘方法page(m)’：返回Page对象，表示第m页的数据，下标从1开始

Page对象

某一页的数据

  • 调用Paginator对象的page()方法返回Page对象，不需要手动构造
  • 属性object_list：返回当前页对象的列表
  • 属性number：返回第几页，从1开始
  • 属性paginator：当前页对应的Paginator对象
  • 方法has_next()：如果有下一页返回True
  • 方法has_previous()：如果有上一页返回True
  • 方法len()：返回当前页面对象的个数
  • 迭代页面对象：访问当前页面中的每个对象

部署

  1. 浏览器：发起请求
  2. Nginx：如果静态文件读取磁盘返回，如果不是静态文件，将请求信息转发uWSGI
  3. uWSGI：接收请求，进行url匹配，找到请求的视图函数，调用执行并返回响应
