模型Model是MVT模式中的M层

ORM

作用：解耦，使代码 低耦合，高内聚
功能：负责对象与关系型数据库之间的交互

管理器Manager

ORM中的核心部分
默认管理器名为：objects

自定义管理器

  • 属性objects：管理器，是Manager类型的对象，管理器负责和数据库交互，自定义管理器就不再生成objects管理器，如books = models.Manager()

    • 自定义管理器用户：

      1. 更改原始查询集

      2. 增加模型类的方法，例如创建对象方法

        类方法写在自定义管理器中，对象方法放在类中

选项（约束字段）

  • null：是否允许为空，针对数据库约束
  • blank：是否允许空白，针对表单验证约束
  • db_column：指定列字段名称，默认为属性名
  • db_index：
  • default：默认值
  • primary_key：主键
  • unique：唯一值，主要用来做重复性检查

关系字段类型

  • ForeignKey：一对多
  • ManyToManyField：多对多
  • OneToOneField：一对一

default值只影响admin中的默认值，并不影响数据库的默认值

模型实例方法

  • str：python对象在终端输出调用的是repr方法，在浏览器输出调用的是str方法
  • save()：将模型对象保存到数据表中
  • delete()：将模型对象从数据表中删除

查询集

两大特性：

  • 惰性执行：创建查询集时不会访问数据库，直到调用才会访问数据库，调用情况包括：迭代、序列化、或if
  • 缓存：查询结果会被存入缓存中

过滤器：

  • all()：返回所有数据
  • filter()：返回满足条件的数据
  • exclude()：返回满足条件之外的数据
  • order_by()：排序

单个值过滤器：

  • get()：返回单个满足条件的徐对象
  • count()：查询总条数
  • aggregate()：聚合
  • exists()：是否有数据，True/False

限制查询集：
类似sql中的limit，但不支持负数
